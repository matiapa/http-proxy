
# Estructura del Request

0                        8
+------------------------+  0
|         VERSION        |
+------------------------+  1
/       PASSPHRASE       /
/                        /
+------------------------+  33
/          ID            /
/                        /
+------+--------+--------+  35
| TYPE | METHOD | LENGTH |
+------+--------+--------+  36
|         LENGTH         |
+---------------+--------+  37
|     LENGTH    |        |
+---------------+--------+  38

donde:

VERSION         número de 1 byte que especifica la versión del servicio

PASSPHRASE      palabra de 32 bytes utilizada para acceder al servicio

ID              número de 2 bytes utilizado para identificar al
                datagrama

TYPE            número de 1 bit utilizado para especificar el tipo de
                solicitud

METHOD          número de 4 bits utilizado para especificar el método
                que se está solicitando

LENGTH          número de 2 bytes utilizado para especificar el tamaño
                del body

# VERSION values
1               Versión actual del servicio

# TYPE values
0               Se pide utilizar un método del tipo retrieve
1               Se pide utilizar un método del tipo modificación de
                configuración

# METHOD values
0               Cuando TYPE está en 0 el método que se pide es TOTAL
                CONNECTIONS, el cual devuelve la cantidad total de
                conexiones historicas. Cuando TYPE es 1 el método que
                se pide es cambiar la cantidad máxima de clientes cuyo
                valor por defecto es 1000 y es el máximo valor posible.

1               Cuando TYPE está en 0 el método que se pide es CURRENT
                CONNECTIONS, el cual devuelve la cantidad total de
                conexiones actuales. Cuando TYPE es 1 el método que se
                pide es cambiar el timeout de conexión. Este valor
                especifica el tiempo máximo en segundos el cual un 
                usuario puede estar inactivo. Para desactivar esta 
                función mandar un -1. Por defecto esta inactivo.

2               Cuando TYPE está en 0 el método que se pide es TOTAL
                SENT, el cual devuelve la cantidad total de bytes que
                se mandaron a través del proxy. Cuando TYPE es 1 el
                método que se pide es cambiar la frequencia (en 
                segundos) de las estadísticas. Para deshabilitarlo 
                mandar un -1.

3               Cuando TYPE está en 0 el método que se pide es TOTAL
                RECIEVED, el cual devuelve la cantidad total de de
                bytes recividos por el proxy. Cuando TYPE es 1 el
                método que se pide es cambiar disectorsEnabled que
                permite extraer credenciales en formato de text/plain.
                Se debe enviar un 0 (desactivar) o un 1 (activar). El
                valor es 1 por defecto.

4               Cuando TYPE está en 0 el método que se pide es ALL, el
                cual retorna todos los valores anteriores en el orden
                anterior en formato de long.

5               Cuando TYPE está en 0 el método que se pide es obtener
                la configuración actual del proxy en el orden de
                anterior en formato de long.
            

# Estructura del Response

0                        8
+------------------------+ 0
|         VERSION        |
+------------------------+ 1
/           ID           /
/                        /
+--------+------+--------+ 3
| STATUS | TYPE | METHOD |
+--------+------+--------+ 4
/         LENGTH         /
/                        /
+------------------------+ 6

donde:

VERSION         número de 1 byte que especifica la versión del servicio

STATUS          número de 2 bits utilizado para especificar el estado 
                de la respuesta

LENGTH          número de 2 bytes utilizado para especificar el tamaño 
                del body
